# JIRA Triage Comment Template

Use this template when posting automated triage analysis comments to JIRA issues.

## Format

```
🤖 Automated Triage Analysis

**Recommended Team:** {{team_mention}}
**Confidence:** {{confidence}}%
**Strategy:** {{strategy}}

**Reasoning:**
{{reasoning}}

**Evidence:**
{{evidence}}

---
_Generated by ACS Triage Workflow_
```

## Variables

- `{{team_mention}}` - JIRA team mention using proper format (see Team Mention Format below)
- `{{confidence}}` - Confidence percentage (e.g., 90)
- `{{strategy}}` - Strategy used (e.g., "Error Signature Match")
- `{{reasoning}}` - One-sentence explanation of the assignment
- `{{evidence}}` - Bulleted list of supporting evidence:
  - Error type
  - File paths
  - Matched signatures
  - Component mappings
  - Similar issue references

## Team Mention Format

Convert GitHub team handles to JIRA team mentions using `reference/jira-team-mappings.md`:

**Format:** `[Team Display Name](https://redhat.atlassian.net/jira/people/team/{team-id}?ref=jira$&src=issue)`

**Examples:**
- `@stackrox/core-workflows` → `[ACS Core Workflows](https://redhat.atlassian.net/jira/people/team/ec74d716-af36-4b3c-950f-f79213d08f71-1813?ref=jira$&src=issue)`
- `@stackrox/scanner` → `[ACS Scanner](https://redhat.atlassian.net/jira/people/team/ec74d716-af36-4b3c-950f-f79213d08f71-743?ref=jira$&src=issue)`

If GitHub team not in mapping, use plain text team name as fallback.

**Reasoning:**

## Usage

Only post comments for issues with confidence ≥80%. Skip low-confidence assignments.
