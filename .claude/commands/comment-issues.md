# /comment-issues - Add Triage Comments to JIRA

## Purpose

Add triage analysis comments to JIRA issues based on team assignment results. Posts structured comments with team recommendation, confidence score, and reasoning.

**⚠️ WARNING:** This command WRITES to JIRA. Unlike other commands, this modifies external state.

## Prerequisites

- `artifacts/acs-triage/issues.json` exists with team assignments
- JIRA MCP write permissions
- `/assign-team` completed successfully

## Process

1. **Read Triage Results**
   - Load `artifacts/acs-triage/issues.json`
   - Filter for issues with team_assignment field

2. **For Each Issue**

   a. **Generate Comment Text**

   Format triage comment:
   ```markdown
   ## 🤖 Automated Triage Analysis

   **Recommended Team:** {assigned_team}
   **Confidence:** {confidence}% ({strategy})
   **Type:** {type}

   ### Analysis
   {reasoning}

   ### Evidence
   {evidence summary - file paths, error patterns, etc.}

   {version_mismatch warning if applicable}

   ---
   *Generated by ACS Triage Workflow • {timestamp}*
   *See full report: [artifacts/acs-triage/report.html]*
   ```

   b. **Post Comment**
   - Use JIRA MCP to add comment to issue
   - Handle rate limiting (max 10 issues/minute)
   - Log success/failure

   c. **Track Results**
   - Count: commented, skipped, failed
   - Store errors for review

3. **Generate Summary**
   - Total issues processed
   - Successfully commented
   - Failures (with reasons)
   - Skipped (low confidence <70%)

## Safety Guardrails

**Confidence Filter:**
- Only comment on issues with confidence ≥70%
- Flag low-confidence issues for manual review

**Dry Run Mode:**
- Default: dry_run = true
- Shows what would be commented without posting
- Require explicit confirmation to post

**Version Mismatch:**
- Add warning banner for version mismatched issues
- Reduce confidence display in comment

## Usage Examples

**Dry run (preview only):**
```
/comment-issues
```

**Actually post comments:**
```
/comment-issues --confirm
```

**Filter by confidence:**
```
/comment-issues --min-confidence=80 --confirm
```

```markdown

**Recommended Team:** @stackrox/core-workflows
**Confidence:** 90% (error_signature)
**Type:** CI_FAILURE

### Analysis
GraphQL schema validation error pattern matches core-workflows team with 90% confidence.

### Evidence
- **Error Type:** GraphQL
- **File Paths:**
  - central/graphql/resolvers/policies.go
- **Error Pattern:** GraphQL schema validation

*Generated by ACS Triage Workflow • 2026-04-27 15:30 UTC*
```

## Success Criteria

After running this command:
- [ ] JIRA issues have triage comments
- [ ] Only high-confidence issues (≥70%) commented
- [ ] Low-confidence issues flagged for manual review
- [ ] Failures logged in comment-results.json
- [ ] No duplicate comments on same issue

## Notes

- **Rate Limiting:** JIRA MCP may have rate limits (10 req/min typical)
- **Idempotency:** Check if triage comment already exists before posting
- **Reversibility:** Comments can be deleted manually in JIRA
- **Audit Trail:** All operations logged in comment-results.json
- **Dry Run Default:** Always preview before posting to prevent accidents
